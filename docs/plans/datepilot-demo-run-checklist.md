# DatePilot Demo Run Checklist

## Goal
Create a repeatable path for demoing and capturing DatePilot without improvising app state every time.

## In-app path
1. Open Home
2. Go to Demo Mode
3. Tap `Load Demo State`
4. Return to Home and capture dashboard
5. Open each tool and tap `Use sample`
6. Run the tool and capture the result view
7. Capture Paywall
8. Capture Settings/Support/Privacy

## Screens to capture
- Home
- Profile Doctor result
- Opener Generator result
- Reply Coach result
- Ask-Out Helper result
- Paywall
- Settings & Support

## Capture hygiene
- ensure no keyboard is open
- use sample content for consistency
- keep the same tone/goal across the run
- avoid half-filled states unless intentionally demonstrating input flow

## Reset path
Use `Reset App State` in Demo Mode when you want to return the app to a clean onboarding state.
