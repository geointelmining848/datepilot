# DatePilot Screenshot Plan

## Goal
Create a clean, store-ready screenshot set for both the App Store and Google Play that communicates the product quickly:
- what DatePilot does
- why it is useful
- why it is different from generic AI chat tools

## Visual style direction
- dark UI
- sharp contrast
- concise headlines
- one core message per screenshot
- avoid cluttered paragraphs
- show actual product screens, not fake marketing abstractions

## Screenshot set (core)

### 1. Home / Dashboard
**Headline:** Better matches. Better conversations.
**Screen:** Home dashboard with tool cards, tone, free uses, and AI mode.
**Purpose:** Establish DatePilot as a focused utility app, not a generic chat app.

### 2. Profile Doctor
**Headline:** Rewrite your profile with stronger hooks
**Screen:** Profile Doctor result view with diagnosis + recommended rewrite.
**Purpose:** Show transformation and immediate value.

### 3. Opener Generator
**Headline:** Send first messages that actually feel natural
**Screen:** Best opener + alternatives.
**Purpose:** Show instant practical value and repeat use case.

### 4. Reply Coach
**Headline:** Get the best next reply
**Screen:** Best reply + alternatives + avoid-this guidance.
**Purpose:** Show the strongest retention loop.

### 5. Ask-Out Helper
**Headline:** Know when to ask — and how
**Screen:** Timing read + low-pressure + confident option.
**Purpose:** Show that the app helps move toward actual outcomes.

### 6. Pro / Upgrade
**Headline:** Unlock more coaching with Pro
**Screen:** Paywall screen.
**Purpose:** Make premium value legible and store-appropriate.

### 7. Settings / Trust
**Headline:** Private. Practical. In your control.
**Screen:** Settings/support/privacy screen.
**Purpose:** Reinforce trust and clarify that DatePilot is a coaching tool, not an automation bot.

## Alternate screenshot candidates
- onboarding screen
- preferences/style screen
- support/privacy screen as a trust-focused final slide

## Asset requirements to prepare later

### Apple App Store
Plan for sizes such as:
- 6.7-inch iPhone
- 6.5-inch iPhone

### Google Play
Plan for:
- phone screenshots in portrait
- feature graphic later if needed

## Copy principles
- one short headline per image
- no cheesy dating language
- no manipulative claims
- no claims of automatic messaging or app integration
- emphasize coaching, rewriting, and guidance

## Capture checklist
Before taking screenshots:
- clean demo content in all fields
- hide obviously placeholder text
- use polished examples
- ensure consistent spacing and theme
- avoid empty states unless intentional

## Next production step
Once native/screenshots are ready to capture, produce:
- headline overlays
- screenshot order
- store-specific crop/export checklist
