# Shipping Checklist

Use this when moving from app concept to implementation plan.

## Goal

Turn a promising app idea into a realistic first build without overbuilding.

## Checklist

### 1. Problem and user
- Can the user and problem be described in one sentence each?
- Is the problem painful enough to matter weekly or monthly?

### 2. MVP scope
- What is the smallest usable product?
- What is explicitly out of scope for v1?
- What fake/manual ops can replace automation initially?

### 3. Monetization
- What is the first revenue mechanism?
- At what point in the user journey does payment happen?
- What would make someone pay in the first week of use?

### 4. Distribution
- Which acquisition channel is primary?
- Is the idea search-driven, social-driven, creator-driven, or community-driven?
- What content or keywords will validate demand fastest?

### 5. Build stack
- Client: Expo/React Native or native iOS?
- Backend: Supabase / Firebase / simple server / no backend?
- Payments: Stripe / IAP / none yet?
- Analytics: what events matter on day one?

### 6. Risk review
- Platform policy risks?
- Data/privacy risks?
- AI cost or latency risks?
- Dependency on brittle third-party APIs?

### 7. Validation plan
- What must be true in 7 days to keep going?
- What metrics or feedback kill the idea early?
- What is the fastest path to real user feedback?

## Output Format

```markdown
SHIPPING CHECKLIST
- App:
- User/problem:
- MVP:
- Out of scope:
- Monetization:
- Distribution:
- Stack:
- Risks:
- 7-day validation plan:
- Go / no-go:
```

## Bias

Prefer shipping a narrow paid utility over a broad ambiguous platform.
